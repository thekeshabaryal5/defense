How the Internet Works

A simple summary from first principles

Core Principles

  1. Direct Link: Connect two computers with a cable to share data.
  2. Local Network: Use a switch so multiple computers can communicate.
  3. Routers: Connect networks together to form the Internet.
  4. IP Address: Every device needs a unique address to be found.
  5. Packets & TCP/IP: Break data into small packets for reliable transfer.

IP Addresses

  • IPv4: 32-bit, ~4.3 billion addresses (e.g., 172.217.16.142).
  • IPv6: 128-bit, almost unlimited (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

Public vs Private IP

Public IP Private IP
Globally unique Local only
Assigned by ISP Assigned by router
Used for servers and websites Used for home/office devices

MAC Address

A unique hardware ID for every network device (e.g., 3C:22:FB:A3:B4:C5).

Port Numbers

Identify specific services on a device (range: 0–65535).

  • Well-Known Ports (0–1023): HTTP (80), HTTPS (443), SSH (22).
  • Registered Ports (1024–49151): Node.js (3000), MongoDB (27017).
  • Dynamic/Ephemeral (49152–65535): Temporary client connections.

DNS (Domain Name System)

Translates human-readable names into IP addresses.

  1. Check browser/OS/router cache.
  2. Ask Recursive Resolver (usually ISP or Google 8.8.8.8).
  3. Resolver queries Root → TLD → Authoritative servers.
  4. Final IP returned to browser.
  5. Browser connects to server using that IP.

Conclusion

The Internet is a network of networks. Using IP addresses, packets, ports, and DNS, billions of devices communicate reliably across the globe.
