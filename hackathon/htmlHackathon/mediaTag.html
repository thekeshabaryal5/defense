HTML Media

First Principle

The web is more than text and images. In the past, video and audio required plugins like Flash or QuickTime. HTML5 introduced <video> and <audio> for native, plugin-free media.

1. The <video> Tag

Basic usage:

<video src="my-video.mp4"></video>

Add controls for play, pause, and volume:

<video src="my-video.mp4" controls></video>

Useful Attributes

  • width and height: set dimensions
  • autoplay: start automatically (often requires muted)
  • loop: restart when finished
  • poster: thumbnail before play

2. The <audio> Tag

Similar to video, but only for sound:

<audio src="song.mp3" controls></audio>

3. Multiple Sources (<source>)

Different browsers support different formats. Use <source> for fallbacks: 

<video controls>
  <source src="movie.webm" type="video/webm">
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

4. Accessibility with <track>

Add captions or subtitles:

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track src="captions_en.vtt" kind="captions" srclang="en" label="English">
</video>

5. Methods of Embedding

Method 1: Using <source>

Best for compatibility. Supports multiple formats.

Method 2: Using src Attribute

Simpler, but only one format. May fail if unsupported.

Comparison

Feature <source> Method src Method
Browser Compatibility Excellent (multiple formats) Good, but limited
Flexibility High Low
Simplicity More complex Very simple
Fallback Yes No
Best Use Public websites Quick tests or internal use

6. HTML Entities

Entities let you display reserved symbols:

  • © → &copy; or &#169;
  • ® → &reg; or &#174;
  • ™ → &trade; or &#8482;
  • < → &lt; or &#60;
  • > → &gt; or &#62;
  • & → &amp; or &#38;

7. Example Footer

<footer>
  <p>Copyright &copy; 2025 My Website. All Rights Reserved.</p>
</footer>
