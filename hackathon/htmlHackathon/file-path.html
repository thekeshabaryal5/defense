File Path system

Step 1: The First Principle (The Fundamental Truth)

The most fundamental truth is this: A website is not one single file.

It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that all need to work together. These files are organized into folders, just like documents on your computer.

Step 2: The Core Problem

If your index.html file needs to display an image named logo.png, how does the HTML file tell the browser where to find it? This is where file paths come in.

Relative File Paths

Relative paths give directions to a file starting from the location of the file you are currently in.We will use this 99% of the time. 

my-website/
├── index.html
├── about.html
│
├── images/
│   ├── logo.png
│   └── hero.jpg
│
└── pages/
    ├── contact.html
    └── terms.html

Scenario 1: Same Folder

<a href="about.html">About Us</a>

Scenario 2: Sub-Folder (Going Down)

<img src="images/logo.png" alt="My Website Logo">

Scenario 3: Parent Folder (Going Up)

<img src="../images/logo.png" alt="My Website Logo">

Absolute File Paths

Absolute file paths give the full URL to a resource. Used for external links. 

<a href="https://www.google.com">Search on Google</a>
<img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

Why not use absolute paths for your own files?

Local computer paths like C:/Users/Arjun/Desktop/my-website/images/logo.png only work on your computer and break on servers. Browsers also prevent HTML from accessing private local files for security reasons.

Summary / Rules of Thumb

  1. For your own files: use Relative Paths.
  2. For other websites’ files: use Absolute Paths.

Windows vs macOS/Linux File Systems

Feature Windows macOS / Linux
Starting Point Drive Letter (C:\) Single Forward Slash (/)
Directory Separator Backslash (\) Forward Slash (/)
Example C:\Users\JohnDoe\Documents\report.docx /Users/johndoe/Documents/report.docx

HTML Boilerplate Code

Every HTML page should start with a boilerplate to give the browser setup info. 

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

Each line (doctype, html, head, meta, title, body) serves a purpose for standards, SEO, mobile optimization, and accessibility.

Why websites work even without boilerplate

Browsers are forgiving and guess missing parts, but this causes quirks mode, broken characters, bad mobile experiences, and poor SEO. Always use a boilerplate.

Multipage Website

Websites usually have multiple pages (Home, About, Contact). To connect them:

  • Use a consistent structure (same header/footer on all pages).
  • Use <a> tags for navigation links.

The <div>, class, and id

<div> is a generic block-level container to group elements. 

<div>
  <img src="avatar.png">
  <h2>Arjun Kumar</h2>
  <p>Loves to code and teach HTML.</p>
</div>

To target groups:

  • id: unique identifier (only once per page).
  • class: reusable label (used on many elements).

The <span>

<span> is an inline container for styling a part of text. 

<p>This is very <span class="highlight">important</span> information.</p>

Semantic Tags

  • <header>: Intro content, usually logo + nav.
  • <footer>: Closing content (copyright, links).
  • <main>: The unique, main content of the page.
