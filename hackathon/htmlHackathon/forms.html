Forms in HTML

Why Forms?

Forms make the web interactive by collecting information from users and sending it to servers. Without them, there would be no logins, searches, or purchases.

The <form> Element

Acts as a container for all input fields and a submit button.

1. The <input> Element

Creates input fields. Behavior changes based on type (e.g., text, password, number, date). 

<input type="text">

2. Labels (<label>)

Gives meaning to input fields. Connected using for (label) and id (input). 

<label for="firstName">First Name:</label>
<input type="text" id="firstName" name="firstName">

3. Submit Button

Submits form data. Better to use <button type="submit"> instead of <input type="submit">.

4. Radio Buttons

Used when only one option can be chosen. All radios in a group share the same name. 

<input type="radio" name="shirtSize" value="small"> Small

5. Checkboxes

Allow multiple selections. All share the same name.

<input type="checkbox" name="toppings" value="mushrooms"> Mushrooms

6. Textarea

For multi-line text input like comments.

<textarea id="comments" name="comments" rows="4" cols="50"></textarea>

7. Dropdown (<select>)

Compact list of options using <option> inside <select>. 

<select name="country">
  <option value="in">India</option>
  <option value="us">USA</option>
</select>

8. Specialized Inputs

  • type="password" → Masks sensitive input.
  • type="number" min="18" max="99" → Restricts numeric input to a range.
  • type="date" → Provides a date picker.

Summary

  • Use <form> to group fields.
  • <label> improves usability & accessibility.
  • name attributes are crucial for identifying submitted data.
  • Use the right input type for better user experience.
